# 新しいgitコミットを作成

## 重要な前提事項

- このタスクはコミットが許可される2つのコマンドのうちの1つ（もう1つは `commit!`）
- はじめに `git add -A` で変更をステージングする
- ステージング済みの差分のみを根拠にコミットメッセージを提案する
- ユーザーの明示的な承認があるまで `git commit -m` を実行しない
- コミットに Claude 共著フッターを追加しない

## コミットメッセージ規約

- 形式: `絵文字 scope: 説明`
- 説明は日本語で簡潔に書く
- scope は変更の主対象を短く表す（例: `docs` `workflow` `claude` `root`）
- プロジェクト固有の固定スコープ規則は使わない

### 絵文字の選択基準

`docs/guides/CODING_STANDARDS.md` の絵文字一覧を参照する。

## 実行手順

1. `git add -A` を実行する
2. `git diff --cached --name-only` でステージング対象をユーザーに提示し、意図しないファイルが含まれていないか確認する
3. 問題がなければ `git diff --cached` で変更内容を把握する
4. `git log -n 100 --oneline` を確認して直近の文体・粒度を合わせる
5. 規約に沿ったコミットメッセージ候補を5件以上、番号付きで提示する
6. ユーザーが番号指定または文面修正で明示承認したら `git commit -m "..."` を実行する

## 提案例

- `📝 docs: コーディング規約のコミット絵文字一覧を更新`
- `🔧 claude: コミットコマンドのメッセージ形式を統一`
- `♻️ workflow: ルール文書の記述を整理`
